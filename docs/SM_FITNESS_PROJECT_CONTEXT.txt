3. CORE FEATURES (CONFIRMED)
--------------------------------------------------------

A. MEMBER MANAGEMENT
   - Add member: full name, mobile, email, date of birth, gender, address
   - Emergency contact name + phone
   - Photo: capture via webcam OR upload from file
     → Compress image client-side (Canvas API) before upload to Supabase Storage
     → Compressed target: ~150KB per photo (not raw 2-3MB)
   - Auto-generate member code: GYM-001, GYM-002...
     → Counter stored in DB (not COUNT(*)) to prevent gaps on deactivation
   - Edit member details
   - Soft deactivate member (never hard delete, set is_active = false)
   - View full member profile page
   - Admin notes field per member

B. MEMBER LIST & SEARCH
   - Paginated list: 20 members per page
   - Search by name OR mobile number (client-side if <500 members)
   - Filter by: all / active / expired / expiring soon
   - Show member photo thumbnail, name, code, plan, expiry status

C. MEMBERSHIP & FEES
   - Assign plan: Monthly (1mo) / Quarterly (3mo) / Half-Yearly (6mo) / Annual (12mo)
   - Fee per membership is CUSTOM — admin enters actual negotiated amount each time
     → Plans only define duration, not a fixed price
   - Early renewal: new membership start_date = old membership end_date + 1 day
     → Auto-calculated in UI, admin cannot enter wrong dates accidentally
   - Multiple active memberships ALLOWED but admin shown a warning:
     "This member already has an active membership until [date]. Continue?"
   - Membership STATUS is always calculated on-read (compare end_date with today IST)
     → NEVER rely on a stored status column for showing active/expired
     → Cron only updates status column for email trigger logic, not for display
   - Membership status values: active / expiring_soon (≤7 days) / expired / cancelled

D. PAYMENTS
   - Two modes:
     1. CASH: Admin collects physically, clicks "Mark as Paid" button
     2. UPI: System shows QR code with amount pre-filled
        → UPI string: upi://pay?pa=PLACEHOLDER_UPI@bank&pn=SM+FITNESS&am=AMOUNT&cu=INR
        → Admin confirms payment after member pays, clicks "Confirm Payment"
   - One-time payment per membership (no partial payments)
   - Auto-generate receipt number: RCP-2026-0001
     → Yearly reset: RCP-2027-0001 on Jan 1
     → Counter stored in DB per year
   - Payment modes: cash / upi
   - Optional UPI reference / UTR number field (admin can note it down)
   - Admin notes field per payment

E. EMAIL NOTIFICATIONS (Nodemailer + Gmail SMTP)
   Trigger                 Email Type          Notes
   -------------------     ----------------    ---------------------------
   Member registered       Welcome email       Sent immediately on registration
   Payment recorded        Receipt email       Styled HTML, sent immediately
   7 days before expiry    Reminder email      Sent by daily cron at 8AM IST
   1 day before expiry     Urgent reminder     Sent by daily cron at 8AM IST
   Day of expiry           Expired notice      Sent by daily cron at 8AM IST

   - All emails logged in email_logs table to prevent duplicates
   - If email fails: logged as status=failed, admin can retry via "Resend Email" button
   - Starting config: developer's Gmail for testing
   - Production: replace with gym owner's Gmail SMTP credentials in env vars
   - Receipt format: styled HTML email (NOT PDF) — gym logo, member name,
     plan, amount, dates, receipt number, payment mode

F. ADMIN DASHBOARD
   - Summary cards: total members, active members, expired, new this month
   - Revenue card: this month total vs last month total
   - Alert banner: members expiring in next 7 days (with count + list)
   - Upcoming renewals table: next 7 days (name, plan, expiry date, action button)
   - Recent payments feed: last 10 payments
   - Quick action buttons: Add Member, Record Payment

G. REPORTS & EXPORT
   - Export full member list as CSV (name, mobile, email, plan, expiry, status)
   - Monthly revenue summary (month-wise totals, payment mode breakdown)
   - "Backup Reminder" notice visible in Reports page:
     "Free tier has no automatic backup. Export member data monthly."

H. AUTH & SECURITY
   - Single admin login: email + password via Supabase Auth
   - Session managed by Supabase (auto token refresh)
   - Protected routes: unauthenticated users redirected to /login
   - Auto logout after 30 minutes of inactivity
   - Supabase RLS (Row Level Security) enabled on ALL tables
     → Policy: only authenticated admin UID can read/write
   - Input validation using Zod on every API route (server-side)
   - Input validation on all forms (client-side, Zod + react-hook-form)

I. CRON JOBS (Vercel Cron — runs at 8AM IST daily)
   Job 1: /api/cron/reminders
     → Query memberships expiring in 7 days → send reminder_7d email (if not sent)
     → Query memberships expiring in 1 day  → send reminder_1d email (if not sent)
     → Query memberships expired today      → send expired email (if not sent)
     → Check email_logs for duplicates before sending
   
   Job 2: /api/cron/ping
     → SELECT 1 from Supabase every 3 days
     → Prevents Supabase free project from pausing after 7 days inactivity

J. PRINT RECEIPT (FALLBACK)
   - "Print Receipt" button on payment confirmation screen
   - Opens browser print dialog with styled receipt layout
   - Covers case where member has no email or email delivery fails

K. UX DETAILS
   - Confirmation dialogs on all destructive actions:
     Deactivate member / Cancel membership / Delete payment record
   - Loading states on all async operations (API calls, photo upload, email send)
   - Toast notifications for success/error on all actions
   - Empty states with helpful messages (no members yet, no payments, etc.)
   - Mobile-responsive layout (admin may use tablet at gym counter)

4. DATABASE SCHEMA (FINAL)
--------------------------------------------------------

TABLE: plans
  id               UUID PRIMARY KEY DEFAULT gen_random_uuid()
  name             TEXT NOT NULL              -- Monthly, Quarterly, Half-Yearly, Annual
  duration_months  INT NOT NULL               -- 1, 3, 6, 12
  is_active        BOOLEAN DEFAULT true
  created_at       TIMESTAMPTZ DEFAULT now()

  SEED DATA:
    ('Monthly', 1), ('Quarterly', 3), ('Half-Yearly', 6), ('Annual', 12)

TABLE: members
  id                        UUID PRIMARY KEY DEFAULT gen_random_uuid()
  member_code               TEXT UNIQUE NOT NULL      -- GYM-001, GYM-002
  full_name                 TEXT NOT NULL
  mobile                    TEXT NOT NULL
  email                     TEXT
  date_of_birth             DATE
  gender                    TEXT                      -- male / female / other
  address                   TEXT
  emergency_contact_name    TEXT
  emergency_contact_phone   TEXT
  photo_url                 TEXT                      -- Supabase Storage URL
  is_active                 BOOLEAN DEFAULT true
  notes                     TEXT
  created_at                TIMESTAMPTZ DEFAULT now()
  updated_at                TIMESTAMPTZ DEFAULT now()

TABLE: member_code_counter
  id       INT PRIMARY KEY DEFAULT 1
  counter  INT DEFAULT 0
  -- Single row, increment atomically when adding member
  -- member_code = 'GYM-' || LPAD(counter::TEXT, 3, '0')

TABLE: memberships
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid()
  member_id     UUID NOT NULL REFERENCES members(id)
  plan_id       UUID NOT NULL REFERENCES plans(id)
  fee_charged   NUMERIC(10,2) NOT NULL    -- custom fee entered by admin
  start_date    DATE NOT NULL
  end_date      DATE NOT NULL
  status        TEXT DEFAULT 'active'     -- active / expired / cancelled
                                          -- NOTE: always recalculate on read, don't trust this alone
  created_by    UUID REFERENCES admins(id)
  created_at    TIMESTAMPTZ DEFAULT now()

TABLE: payments
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid()
  membership_id   UUID NOT NULL REFERENCES memberships(id)
  member_id       UUID NOT NULL REFERENCES members(id)
  amount          NUMERIC(10,2) NOT NULL
  payment_mode    TEXT NOT NULL           -- cash / upi
  payment_date    TIMESTAMPTZ DEFAULT now()
  upi_ref         TEXT                    -- optional UTR / reference number
  receipt_number  TEXT UNIQUE NOT NULL    -- RCP-2026-0001
  email_sent      BOOLEAN DEFAULT false
  notes           TEXT
  created_by      UUID REFERENCES admins(id)
  created_at      TIMESTAMPTZ DEFAULT now()

TABLE: receipt_counter
  id     INT PRIMARY KEY DEFAULT 1
  year   INT NOT NULL
  counter INT DEFAULT 0
  -- Reset counter to 0 when year changes
  -- receipt_number = 'RCP-' || year || '-' || LPAD(counter::TEXT, 4, '0')

TABLE: email_logs
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid()
  member_id   UUID NOT NULL REFERENCES members(id)
  type        TEXT NOT NULL    -- welcome / receipt / reminder_7d / reminder_1d / expired
  sent_to     TEXT NOT NULL    -- email address at time of sending
  status      TEXT NOT NULL    -- sent / failed
  error_msg   TEXT             -- error message if failed
  sent_at     TIMESTAMPTZ DEFAULT now()
  membership_id UUID REFERENCES memberships(id)  -- for reminder/expiry types

TABLE: admins
  id          UUID PRIMARY KEY      -- matches Supabase Auth user id
  name        TEXT NOT NULL
  email       TEXT NOT NULL
  role        TEXT DEFAULT 'owner'  -- owner / staff (staff for future use)
  created_at  TIMESTAMPTZ DEFAULT now()

SUPABASE STORAGE:
  Bucket name: member-photos
  Access: private (only accessible via signed URLs)
  Path pattern: members/{member_id}/photo.jpg
  Max file size: 2MB (after compression should be ~150KB)

8. CRITICAL IMPLEMENTATION NOTES
--------------------------------------------------------

TIMEZONE (MOST IMPORTANT):
- All date comparisons must use IST (UTC+5:30)
- Use date-fns-tz: toZonedTime(utcDate, 'Asia/Kolkata')
- When storing dates: store as DATE (not TIMESTAMPTZ) for membership start/end
- Cron scheduled at 2:30 AM UTC = 8:00 AM IST

MEMBERSHIP STATUS LOGIC:
  const today = toZonedTime(new Date(), 'Asia/Kolkata')
  if end_date < today           → 'expired'
  if end_date <= today + 7 days → 'expiring_soon'
  else                          → 'active'
  NEVER display status from DB column alone

PHOTO UPLOAD FLOW:
  1. Webcam captures frame OR user selects file
  2. Compress to <200KB using Canvas API (imageCompress.ts)
  3. Upload compressed blob to Supabase Storage: members/{id}/photo.jpg
  4. Get public/signed URL, save to members.photo_url

MEMBER CODE GENERATION:
  1. BEGIN transaction
  2. UPDATE member_code_counter SET counter = counter + 1 RETURNING counter
  3. Format: 'GYM-' + counter.toString().padStart(3, '0')
  4. COMMIT
  (atomic — no race conditions, no gaps)

RECEIPT NUMBER GENERATION:
  Same pattern with receipt_counter table
  Check if current year matches stored year; if not, reset counter to 0
  Format: 'RCP-' + year + '-' + counter.toString().padStart(4, '0')

EARLY RENEWAL DATE CALCULATION:
  newStartDate = addDays(existingMembership.end_date, 1)
  newEndDate = addMonths(newStartDate, plan.duration_months)
  Show both dates to admin before confirmation

MULTIPLE MEMBERSHIP WARNING:
  Before creating membership, query:
    SELECT * FROM memberships
    WHERE member_id = ? AND end_date >= today
  If results found, show warning modal — don't block, just warn

SUPABASE RLS POLICIES (apply to ALL tables):
  CREATE POLICY "admin_only" ON table_name
    FOR ALL USING (auth.uid() = (SELECT id FROM admins LIMIT 1));

EMAIL DUPLICATE PREVENTION:
  Before sending any cron email, query email_logs:
    SELECT id FROM email_logs
    WHERE member_id = ? AND type = ? AND membership_id = ?
    AND DATE(sent_at) = today
  Only send if no record found

FACE-API.JS MODELS:
  - Place all model files in /public/models/
  - Load ONLY when PhotoCapture component mounts
  - Use dynamic import: const faceapi = await import('@vladmandic/face-api')
  - Don't import at page level — wastes 6MB on every page

AUTO LOGOUT:
  Track last activity timestamp in localStorage
  On each route change, check if (now - lastActivity) > 30 minutes
  If yes, call supabase.auth.signOut() and redirect to /login
  Reset lastActivity on any user interaction
